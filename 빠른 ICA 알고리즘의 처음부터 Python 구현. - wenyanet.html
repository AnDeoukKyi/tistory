Python에서 처음부터 ICA (Independent Component Analysis) 구현

이것은 빠른 ICA (Independent Component Analysis) 알고리즘 구현에 대한 Python Jupyter Notebook for the Medium 기사 입니다.

ICA는 신호의 선형 혼합을 기본 독립 구성 요소로 분해하는 효율적인 기술입니다. 이 방법이 사용되는 전형적인 예는 이미지의 노이즈 감소, 시계열 데이터에서 아티팩트 제거 또는 재무 데이터의 구동 구성 요소 식별입니다.

여기에서는 먼저 필요한 라이브러리를 가져 와서 ICA 구현을 개발하고 테스트하는 데 사용할 장난감 신호를 만드는 것으로 시작합니다.

장난감 신호 생성

행렬 A에 의해 혼합 될 몇 가지 독립 신호를 생성하는 것으로 시작합니다. 독립 소스 신호는 (1) 사인파, (2) 톱니 신호 및 (3) 랜덤 노이즈 벡터입니다. A로 내적을 계산 한 후 이러한 소스 신호의 세 가지 선형 조합을 얻습니다.

# Set a seed for the random number generator for reproducibility

Interlude : 가우스 신호와 비 가우시안 신호 비교

위에서 이미 언급했지만 ICA가 관찰 된 신호 x가 작동하려면 독립적 인 구성 요소 의 선형 조합 이어야합니다 . 선형성은 선형 인 생성 모델을 따릅니다. 독립성은 신호 x1에 신호 x2에 대한 정보가 포함되어 있지 않음을 의미합니다. 이로부터 두 신호는 상관 관계가없고 공분산이 0입니다. 그러나 두 신호가 상관 관계가 없다고해서 두 신호가 독립적임을 자동으로 의미하지는 않습니다.

충족해야 할 세 번째 전제 조건은 비 가우시 아성입니다.독립적 인 소스 신호의. 왜 그런 겁니까? 두 개의 독립적 인 비 가우시안 신호의 결합 밀도 분포는 정사각형에서 균일합니다. 이 두 신호를 직교 행렬과 혼합하면 이제 더 이상 독립적이지 않고 평행 사변형에서 균일 한 분포를 갖는 두 신호가 생성됩니다. 즉, 혼합 신호 중 하나의 최소값 또는 최대 값에 있으면 다른 신호의 값을 알고 있습니다. 따라서 그들은 더 이상 독립적이지 않습니다. 두 개의 가우시안 신호로 동일한 작업을 수행하면 다른 결과가 발생합니다. 소스 신호의 결합 분포는 완전히 대칭이며 혼합 신호의 결합 분포도 마찬가지입니다. 따라서 계산하려는 역인 혼합 행렬에 대한 정보가 포함되어 있지 않습니다. 이 경우 ICA 알고리즘이 실패합니다.

아래 코드는 가우스 신호와 비가 우스 신호 간의 이러한 차이점을 보여줍니다.

전처리 기능

독립 구성 요소에 대한 최적의 추정치를 얻으려면 데이터의 일부 전처리를 수행하는 것이 좋습니다. 다음에서는 가장 중요한 두 가지 전처리 기술에 대해 자세히 설명합니다.

여기서 논의 할 첫 번째 전처리 단계는 센터링 입니다. 이것은 입력 X에서 평균을 간단히 빼는 것입니다. 결과적으로 중앙 혼합 신호는 0 평균을 갖게되며 이는 소스 신호 s도 평균이 0임을 의미합니다. 이는 ICA 계산을 단순화하고 나중에 평균을 다시 추가 할 수 있습니다.

def center(x):
    mean = np.mean(x, axis=1, keepdims=True)
    centered =  x - mean
    return centered, mean

두 번째 전처리 기법의 경우 공분산을 계산해야합니다. 그러니 빨리 정의합시다.

def covariance(x):
    mean = np.mean(x, axis=1, keepdims=True)
    n = np.shape(x)[1] - 1
    m = x - mean

    return (m.dot(m.T))/n

두 번째 전처리 방법은 미백 이라고 합니다. 여기서 목표는 신호 간의 잠재적 인 상관 관계가 제거되고 분산이 동일한 단일성을 갖는 방식으로 관찰 된 신호 X를 선형으로 변환하는 것입니다. 결과적으로 백색 신호의 공분산 행렬은 단위 행렬과 동일합니다.

def whiten(x):
    # Calculate the covariance matrix
    coVarM = covariance(X)

    # Single value decoposition
    U, S, V = np.linalg.svd(coVarM)

    # Calculate diagonal matrix of eigenvalues
    d = np.diag(1.0 / np.sqrt(S))

    # Calculate whitening matrix
    whiteM = np.dot(U, np.dot(d, U.T))

    # Project onto whitening matrix
    Xw = np.dot(whiteM, X)

    return Xw, whiteM

빠른 ICA 알고리즘 구현

이제 실제 ICA 알고리즘을 살펴볼 때입니다. 위에서 설명한 것처럼 ICA 알고리즘이 작동하기위한 한 가지 전제 조건은 소스 신호가 가우시안이 아니라는 것입니다. 따라서 ICA의 결과는 가능한 한 가우시안이 아닌 소스를 반환해야합니다. 이를 달성하려면 가우시안 성 측정이 필요합니다. 한 가지 방법은 첨도이며 여기에서 사용할 수 있지만 다른 방법이 더 효율적인 것으로 입증되었습니다. 그럼에도 불구하고 우리는이 노트의 끝에서 첨도를 살펴볼 것입니다. 그러나 실제 알고리즘의 경우 방정식 g 및 g '를 사용합니다.

이 방정식은 네전 트로피의 근사를 허용 하며 고정 소수점 반복 체계를 기반으로하는 아래 ICA 알고리즘에서 사용됩니다 .

따라서 위의 내용에 따르면 각 구성 요소의 가중치에 대해 무작위로 추측해야합니다. 랜덤 가중치와 혼합 된 신호의 내적은 두 함수 g 및 g '로 전달됩니다. 그런 다음 g에서 g '의 결과를 빼고 평균을 계산합니다. 결과는 새로운 가중치 벡터입니다. 다음으로 새로운 가중치 벡터를 노름으로 직접 나누고 가중치가 더 이상 변경되지 않을 때까지 위를 반복 할 수 있습니다. 그것은 잘못된 것이 아닙니다. 그러나 여기서 직면 한 문제는 두 번째 구성 요소의 반복에서 첫 번째 반복에서와 동일한 구성 요소를 식별 할 수 있다는 것입니다. 이 문제를 해결하기 위해 우리는 이전에 식별 된 가중치에서 새 가중치를 상호 연관시켜야합니다. 이것은 가중치를 업데이트하고 표준으로 나누는 단계에서 일어나는 일입니다.

def fastIca(signals,  alpha = 1, thresh=1e-8, iterations=5000):
    m, n = signals.shape

    # Initialize random weights
    W = np.random.rand(m, m)

    for c in range(m):
            w = W[c, :].copy().reshape(m, 1)
            w = w / np.sqrt((w ** 2).sum())

            i = 0
            lim = 100
            while ((lim > thresh) & (i < iterations)):

                # Dot product of weight and signal
                ws = np.dot(w.T, signals)

                # Pass w*s into contrast function g
                wg = np.tanh(ws * alpha).T

                # Pass w*s into g prime
                wg_ = (1 - np.square(np.tanh(ws))) * alpha

                # Update weights
                wNew = (signals * wg.T).mean(axis=1) - wg_.mean() * w.squeeze()

                # Decorrelate weights              
                wNew = wNew - np.dot(np.dot(wNew, W[:c].T), W[:c])
                wNew = wNew / np.sqrt((wNew ** 2).sum())

                # Calculate limit condition
                lim = np.abs(np.abs((wNew * w).sum()) - 1)

                # Update weights
                w = wNew

                # Update counter
                i += 1

            W[c, :] = w.T
    return W

첨도

마지막으로 마지막으로 신호의 첨도를 확인하겠습니다. 첨도는 데이터의 네 번째 모멘트이며 분포의 "꼬리"를 측정합니다. 정규 분포의 값은 3이고, 장난감 데이터에서 사용한 것과 같은 균일 분포는 첨도 <3입니다. 아래 코드에서 볼 수 있듯이 Python의 구현은 간단합니다. 데이터. 첫 번째 모멘트는 평균, 두 번째 모멘트는 분산, 세 번째 모멘트는 왜도, 네 번째 모멘트는 첨도입니다. 여기서 3은 정규 분포의 첨도가 0이되도록 네 번째 모멘트에서 뺍니다.

# Calculate Kurtosis

def kurt(x):
    n = np.shape(x)[0]
    mean = np.sum((x**1)/n) # Calculate the mean
    var = np.sum((x-mean)**2)/n # Calculate the variance
    skew = np.sum((x-mean)**3)/n # Calculate the skewness
    kurt = np.sum((x-mean)**4)/n # Calculate the kurtosis
    kurt = kurt/(var**2)-3

    return kurt, skew, var, mean

다음에서 볼 수 있듯이 모든 혼합 신호의 첨도는 ≤ 1 인 반면, 복구 된 모든 독립 구성 요소는 첨도가 1.5이므로 소스보다 가우스가 적습니다. ICA가 비 가우시안 성을 최대화하려고 시도하기 때문에 이것이 사실입니다. 또한 위에서 언급 한 비 가우시안 신호의 혼합이 소스보다 가우시안이 더 많다는 사실을 잘 보여줍니다.
